## What is Logos Storage?

Logos Storage is the storage layer in the Logos Stack. It currently provides a basic filesharing layer — not unlike IPFS or BitTorrent — which allows users to publish and share files with one another over a decentralized network.

Moving forward, Logos Storage will enable full provider and downloader anonymity, as well as anonymous persistence. Check out [our roadmap](https://roadmap.logos.co/storage/roadmap/) for more details.

## Using Logos Storage

Logos Storage can be used in the following ways:

- **Logos Storage UI App** — A [desktop application](https://github.com/logos-co/logos-storage-ui) for uploading and downloading files over the Logos Storage network. This is the easiest way to get started.
- **Logos Storage Module API** — A high-level [C++ API](https://github.com/logos-co/logos-storage-module) for building applications on top of Logos Storage. It underpins the UI App.
- **libstorage** — A lower-level [C API](https://github.com/logos-co/logos-storage-nim) that sits under the Storage Module. Use this to build bindings for other languages. Bindings for [Go](https://github.com/logos-storage/logos-storage-go), [Rust](https://github.com/nipsysdev/storage-rust-bindings), and simplified [C](https://github.com/logos-storage/easylibstorage) are available.
